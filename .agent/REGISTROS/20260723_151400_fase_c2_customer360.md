# Registro Técnico - Fase C.2: Customer 360 Backend Inicial
**Data/Hora**: 2026-07-23 15:14:00 (Local)

## 1. Resumo da Implementação
Concluímos a implementação do backend consolidado de Customer 360 para a aba **Clientes (CRM)** do Painel Admin. O endpoint `customers_summary` em `api_licenca_ml.php` foi expandido e enriquecido para agregar dados de três fontes distintas de forma robusta e segura.

## 2. Fontes Efetivamente Agregadas
1.  **Licenças Locais** (`database_licenses_secure.json`): Base de dados canônica local usada para agrupar clientes por e-mail ou por chave de licença (fallback seguro quando sem e-mail).
2.  **Leads de Evolução** (`api_data/evolution_leads.json`): Leitura e merge dinâmico por e-mail/chave de licença, registrando interesse comercial, WhatsApp e consentimento.
3.  **Dados SaaS/Provisionados** (Supabase): Integração via cURL ao Supabase baseada nas credenciais existentes. Tratamento de fallback completo caso a API esteja indisponível.

## 3. Novos Campos Adicionados ao Payload
Para cada registro na tabela de clientes, o payload de `customers_summary` agora inclui:
*   `customer_key`: Chave primária de consolidação (e-mail ou chave de licença).
*   `sources`: Array com as fontes mapeadas para o cliente (`["licenses", "evolution_leads", "saas"]`).
*   `licenses_count`: Quantidade de licenças locais + SaaS associadas.
*   `leads_count`: Quantidade total de leads associados.
*   `open_leads_count`: Contagem de leads com status ativo no funil (`novo`, `contatado`, `proposta_enviada`).
*   `last_lead_at`: Data/hora do último lead registrado.
*   `last_lead_status`: Status do último lead.
*   `last_interest_type`: Tipo de interesse do lead (`plan_upgrade`, `feature_interest`).
*   `last_target_plan_code`: Código do plano alvo desejado.
*   `customer_whatsapp`: Telefone de WhatsApp coletado no lead.
*   `customer_name`: Nome de exibição principal coletado.
*   `contact_consent`: Flag booleana indicando consentimento para contato.
*   `has_saas`: Booleana indicando se o cliente possui tenants criados no Supabase.
*   `saas_tenants_count`: Quantidade de tenants no SaaS.
*   `trial_count`: Quantidade de licenças locais/SaaS do tipo trial.
*   `active_count` / `blocked_count` / `pending_count`: Distribuição detalhada do status das licenças.
*   `relationship_stage`: Estágio consolidado do cliente (`lead`, `trial`, `customer_local`, `customer_saas`, `mixed`).
*   `data_completeness`: Grau de preenchimento cadastral (`complete`, `partial`, `license_only`, `lead_only`).
